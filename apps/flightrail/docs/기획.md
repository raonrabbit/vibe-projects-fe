# Flightrail 기획서

공부/작업 시간을 비행 여행으로 시각화하는 개인 생산성 도구.

---

## 컨셉

비행기를 타고 여행을 떠나는 것처럼 공부 시간을 기록한다. 시간이 쌓일수록 세계 지도 위에 비행 궤적이 남는다.

---

## 모드

모드 구분 없이 단일 플로우. 여행을 계획할 때 목표 시간을 설정하는 방식만 존재한다.

---

## 핵심 UX 흐름

### 세션 시작

1. 홈 화면 "출발하기" 버튼
2. 모달: 주제(태그) 선택/입력 → 출발지 선택 → 목표 시간 입력 → hard_stop 옵션 설정
3. "탑승" 버튼 → 타이머 화면 전환

### 출발지 선택

- 기본값: 마지막 도착 공항
- 첫 번째 후보: ICN (인천국제공항) 고정 노출
- 나머지: 국가/공항명 검색

### hard_stop 옵션

여행 계획 시 설정하는 토글. "딱 N시간만 하고 싶다"는 경우 켠다.

| 설정      | 동작                                                    |
| --------- | ------------------------------------------------------- |
| 끔 (기본) | 목표 시간 도달 후에도 계속 비행 가능 (연착 상태로 전환) |
| 켬        | 목표 시간 도달 시 자동 착륙 처리                        |

### 일시정지 (Pause)

- 3D 창밖 뷰에 블러 + 어두운 오버레이
- 타이머 정지 (공중에 비행기 멈춤), 재개 버튼 노출
- 일시정지 시간은 비행 시간에 포함하지 않음
- 일시정지 상태로 브라우저/탭 닫아도 세션은 DB에 보존됨 (미완료 여행으로 남음)
- 미완료 여행은 홈 화면에서 이어서 탑승 가능, 최대 10개 보존

### 착륙 전 안내

- 목표 시간 **N분 전** (5분 or 10분, 사용자 설정) 기장 안내방송 오디오 재생
- 목표 도달 시 브라우저 알림 + 연착 여부 표시
- hard_stop이 켜져 있으면 이 시점에 자동 착륙

### 3D 지도 씬 — 목표 시간 도달 후 동작

비행기는 멈추지 않고 계속 난다. 목표 시간은 안내방송/알림 이벤트 트리거용 마일스톤이며,
도착 공항은 세션 종료 시점의 실제 공부 시간으로 계산한다.

### 세션 종료 상태

| 상태                | 조건                            | 설명                           |
| ------------------- | ------------------------------- | ------------------------------ |
| `ontime`            | 실제 시간이 계획의 80~120% 이내 | 대략 계획대로 완주             |
| `delayed`           | 실제 시간 > 계획의 120%         | 계획보다 더 날음               |
| `emergency_landing` | 실제 시간 < 계획의 80%          | 계획보다 일찍 멈춤 (이유 불문) |

연착(delayed)이 꼭 나쁜 것도, 비상착륙(emergency_landing)이 꼭 나쁜 것도 아니다. 히스토리에 사실 그대로 기록한다.

## 3D 씬

### 타이머 화면: 1인칭 창밖 뷰

- React Three Fiber (R3F) + Three.js
- 창문 프레임 + 창밖 하늘/구름 씬
- **시간 기반 변화**: 로컬 현재 시간 기준 (새벽/오전/낮/저녁/밤)
- **모드 설정** (4가지): 자동(로컬시간) / 자동(비행위치 현지시간) / 낮 고정 / 밤 고정
  - 비행위치 현지시간 계산: 목적지 공항 UTC offset 하드코딩 (API 불필요)
- Pause 시: 블러 + 어두운 오버레이

### 홈/기록 화면: 3D 지도 씬

- R3F 3D 공간 안에 **D3.js 세계지도를 평면(Plane mesh)** 으로 배치
  - D3 → Canvas 렌더링 → Three.js 텍스처
- **3D 비행기 모델** (단순 기하학 형태, 회색)이 지도 위를 이동
- 비행 경로는 출발지~목적지 호(arc) 곡선으로 표현
- **지도 낮/밤 모드**: 커스텀 텍스처 (도시 불빛 포함)
- **카메라**: 기본은 비행기 추적 시네마틱, 드래그하면 수동 OrbitControls 전환

## 데이터 모델

```ts
interface Session {
  id: string;
  user_id: string;
  subject_id: string;

  departure_airport: string; // IATA 코드 (예: "ICN")
  destination_airport: string | null; // 종료 시점에 실제 accumulated_seconds 기준 계산, 진행 중엔 null

  started_at: string; // ISO timestamp
  ended_at: string | null; // 진행 중엔 null

  planned_duration: number; // 초 단위
  accumulated_seconds: number; // 실제 공부 시간 누계 (일시정지 시 업데이트)
  hard_stop: boolean; // true면 planned_duration 도달 시 자동 종료

  resumed_at: string | null; // 현재 타이머 시작 시각, 일시정지 중이면 null

  arrival_status: "ontime" | "delayed" | "emergency_landing" | null; // 진행 중엔 null
}

interface Subject {
  id: string;
  user_id: string;
  name: string; // 태그 이름 (예: "알고리즘")
  color?: string;
}
```

### 현재 경과 시간 계산

```ts
// 비행 중
const current = accumulated_seconds + (Date.now() / 1000 - resumed_at_unix);

// 일시정지 중
const current = accumulated_seconds;
```

## 목적지 계산 알고리즘

1. 주요 공항 ~100개 JSON 하드코딩 (IATA코드, 위도/경도, UTC offset, 국가명)
2. 공부 시간(초) → 비행 시간(시간)으로 1:1 매핑
3. 비행 시간 × 900km/h = 도달 가능 거리(km)
4. 출발 공항에서 해당 거리와 가장 가까운 공항 선택 (Haversine 공식)
5. 30분 미만 (~270km 이내): 경비행기 구경 컨셉, 국내 단거리 공항 배정
