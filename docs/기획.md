# Frontend/AI 뉴스 큐레이션 웹앱 기획서

## 1) 프로젝트 개요
- **프로젝트명(가칭)**: Dev & AI News Korea
- **목표**: 한국어 친화적으로 Frontend/AI 뉴스를 한곳에서 모아보고, 사용자 맞춤 알림(Notification)과 키워드 기반 탐색을 제공하는 웹앱 구축
- **개발 형태**: 1인 개발 (풀스택)
- **핵심 가치**
  - 흩어진 기술 뉴스를 빠르게 수집/소비
  - 핫 키워드 중심으로 트렌드 파악
  - 웹앱 푸시 알림으로 중요한 소식 즉시 확인

## 2) 타겟 사용자
- Frontend 개발자
- AI/LLM 관련 기술 동향을 팔로우하는 개발자 및 기획자
- 매일 뉴스를 직접 찾기보다 큐레이션/요약을 선호하는 사용자

## 3) 문제 정의
- 여러 매체를 직접 방문해야 해서 정보 탐색 비용이 높음
- 영어권 중심 뉴스가 많아 한국어 사용자에게 진입장벽 존재
- 어떤 키워드가 현재 뜨는지 한눈에 파악하기 어려움
- 중요한 소식을 놓쳐도 즉시 인지하기 어려움

## 4) 제품 목표 (Goal)
- 한국어 중심으로 Frontend/AI 뉴스 탐색 경험 제공
- 키워드 분석을 통해 트렌드 발견 기능 제공
- Google OAuth 기반 간편 로그인으로 개인화 기능 제공
- 웹앱 Notification으로 실시간성 강화

### 5.2 한국어 친화 UX

#### 비한국어 콘텐츠 처리
- **기본 노출**: LLM으로 생성한 한국어 요약 제공
- **원문/번역 토글**: 기사 상세 페이지에서 원문 ↔ 한국어 번역 전환 가능
- **GitHub 릴리즈 노트**: 영문 원본 + 한국어 요약 병행 제공

#### LLM 처리 파이프라인 (단일 API 호출로 비용 최소화)
기사 1건 수집 시 아래를 **한 번의 LLM 호출**로 처리:
1. 한국어 요약 생성 (3~5줄)
2. 핵심 키워드 추출 (5~10개)
3. 카테고리 분류

> 비용 목표: 월 $30 이하 / 캐싱 및 배치 처리 적용 필수

### 5.3 키워드 분석/트렌드 기능

#### 키워드 추출 방식
- **LLM 기반 추출**: `React Server Component`, `hydration`, `edge runtime` 같은 Frontend 전문 용어를 맥락 기반으로 정확하게 추출
- 기사 수집 시 요약/번역과 **단일 API 호출로 통합 처리**

#### 급상승 키워드 기준
아래 두 조건을 **동시 충족** 시 급상승으로 분류:
- 전일 대비 등장 횟수 N% 이상 증가 (임계값은 데이터 축적 후 튜닝)
- 24시간 내 최소 N건 이상 기사에서 언급 (임계값은 데이터 축적 후 튜닝)

#### 기간별 키워드 랭킹
- 24시간, 7일, 30일 기준 핫 키워드 랭킹
- 키워드 클릭 시 관련 기사 목록 필터링
- 급상승 키워드 별도 섹션 제공

| 유형 | 설명 |
|------|------|
| 구독 키워드 | 구독한 키워드가 포함된 새 기사 발행 시 |

- **세분화 설정**: 마이페이지에서 알림 유형별 켜기/끄기
- **클릭 동작**: 해당 기사 또는 키워드 모음 페이지로 이동

## 7) 사용자 시나리오
1. 사용자가 웹앱 접속 → 비로그인 상태로 뉴스/키워드 트렌드 열람 가능
2. Google 로그인 후 관심 키워드 및 GitHub 저장소 구독 설정
3. 홈에서 최신 뉴스, GitHub 릴리즈, 핫 키워드 확인
4. 특정 키워드 클릭 → 관련 기사 모아보기
5. 기사 상세에서 한국어 요약 ↔ 원문 토글로 전환
6. 알림 허용 시, 구독 키워드 신규 기사 / GitHub 릴리즈 / 급상승 키워드 알림 수신

## 8) 정보 구조(IA)
- `/` 홈: 최신 뉴스 + GitHub 릴리즈 + 핫 키워드 요약
- `/news` 뉴스 목록: 카테고리/기간/출처 필터
- `/releases` GitHub 릴리즈 목록: 저장소별/날짜별 필터
- `/keywords` 키워드 트렌드: 랭킹/급상승/연관 기사
- `/article/:id` 기사 상세: 한국어 요약 ↔ 원문 토글
- `/mypage` 내 설정: 관심 키워드/구독 저장소/알림 유형별 설정/계정
- `/login` OAuth 로그인 페이지

## 10) 기술 구성

### Frontend
- **Framework**: Next.js (App Router) + TypeScript
- **Styling**: Tailwind CSS + 자체 디자인 시스템
- **상태관리**: Zustand 또는 Jotai
- **인증**: NextAuth.js (Google OAuth)

### Backend/API
- **Framework**: Spring Boot + Java
- **ORM**: Spring Data JPA + Hibernate
- **Queue/Scheduler**: Spring Scheduler + Quartz

### 데이터베이스
- **Primary DB**: PostgreSQL
- **Cache**: Redis (키워드 랭킹, API 응답 캐싱)

### 외부 연동
- **LLM API**: Claude API (요약/번역/키워드 추출 단일 호출)
- **GitHub API**: 릴리즈 감지
- **RSS Parser**: 국내외 블로그 수집

### 알림
- Web Push (Service Worker + Push API + VAPID)

### 배포
- **Frontend**: Vercel (무료 티어)
- **Backend**: Railway 또는 Fly.io (Spring Boot 메모리 특성상 유료 필요, ~$5~10/월)
- **DB**: Supabase 또는 Neon (PostgreSQL 무료 티어)
- **Redis**: Upstash (무료 티어)

### 디자인
- **Figma MCP + Claude Code 연동**으로 디자인→코드 변환 워크플로우
